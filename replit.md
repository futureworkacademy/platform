# The Future of Work - Business Simulation Game

## Overview
This web-based business simulation game provides graduate students with an engaging platform for strategic decision-making over a semester, focusing on AI adoption, employee anxiety, and workforce transformation. Teams compete on financial performance and cultural health, offering experiential learning for future business leaders. The platform addresses the market need for AI integration and workforce management training, preparing leaders for complex decision-making in an AI-driven world. The project aims to become a leading educational tool in business strategy, known for its innovative approach to simulating real-world challenges in an AI-driven economy.

## System Architecture

**UI/UX Decisions:**
The design is inspired by the Bloomberg Terminal, offering a sophisticated and premium feel with sharp lines and geometric precision. It uses Shadcn/UI, Radix primitives, and Recharts for data visualization. Styling is managed with Tailwind CSS, featuring a custom color palette with Corporate Navy (#1e3a5f) as the primary brand color and Growth Green (#22c55e) as an accent. Fonts include IBM Plex Sans, Inter, and Roboto Mono. The brand emphasizes an authoritative, forward-thinking, and sophisticated tone with clean, minimal visuals. Public-facing pages use an Apple-style scrolling showcase design with full-viewport sections, AI-generated hero/feature images, and fade-in animations. The brand logo is a Human+AI figure (navy silhouette with circuit board patterns dissolving into green traces), rendered via a shared `BrandLogo` component (`client/src/components/brand-logo.tsx`) with a dual-icon strategy: `variant="horizontal"` (default) uses a head-only crop (`fwa_icon_head.png`) for headers, sidebars, and PDFs where size is constrained; `variant="vertical"` uses the full bust (`favicon.png`) for hero sections and centered displays. The head crop is stored at `client/public/logo-head.png` and `attached_assets/generated_images/fwa_icon_head.png`. The full bust is at `attached_assets/favicon.png` and `client/public/logo.png`. Server-rendered pages (weekly and grading) reference `/logo-head.png` in their headers. PDF headers use the head crop at 28×12.5mm (rendered at 400×180px canvas resolution) via `loadLogoForPdf()` in `pdf-utils.ts`.

**Technical Implementations:**
The application uses React and TypeScript for the frontend, with Wouter for routing and TanStack Query for data fetching. The backend is Express.js, with PostgreSQL as the database and Drizzle ORM. Authentication uses Replit OIDC Auth with connect-pg-simple for session management. The system supports a three-tier role hierarchy (Super Admin, Class Admin, Student) and multi-tenancy.

The core simulation loop includes dashboards, briefings, decision-making, analytics, and leaderboards, with dual scoring for financial and cultural performance. Students can attach up to 5 image visualizations (PNG, JPEG, WebP) per decision response to strengthen their analysis; attachments are stored in object storage, displayed in student/instructor views, and evaluated by GPT-4o vision when present (text-only responses use gpt-4.1-nano). No "optional" language should appear near image upload or visualization attachment features — visualizations directly improve Evidence Quality and Reasoning Coherence rubric scores. The Student Guide (`/guides/student#scoring`) includes a dedicated "How the AI Evaluator Works" section explaining the AI grading pipeline, vision-based chart evaluation, and score optimization strategies. Key features include LLM-powered grading for essay responses with structured 3-part sub-questions and a 4-criterion rubric. The rubric criteria are consistently defined across schema and grading services, with explicit scoring calibration guidance for LLM evaluation. Optional curved scoring based on Z-score normalization is available. Persistent activity logging tracks over 10 event types. Configurable student enrollment supports .edu email verification and SMS notifications. Dedicated admin dashboards provide simulation lifecycle management, multi-scenario modules, and a content editor with AI enhancements. Detailed week results and LLM-evaluated essay scores are provided, and intel engagement bonuses encourage article interaction.

The system includes 17 character profiles with AI-generated headshots, rich bios, and relationship mapping, influencing simulation outcomes through quantifiable traits (influence, hostility, flexibility, risk tolerance). These traits affect decision difficulty, LLM grading context injection, and advisor warnings, with triggered voicemail notifications. A "Phone-a-Friend" advisor system offers AI-generated guidance from 9 specialized advisors. Multimedia content support for briefings includes transcripts and engagement tracking. A 3-tier difficulty framework (Introductory/Standard/Advanced) allows for customizable simulation experiences.

A self-service demo system offers 30-day evaluator access with a hybrid guided tour and a Gemini-powered Q&A assistant. Privacy Mode enables anonymous student enrollment. A unified Role Preview System allows Super Admins to experience the platform as an Educator or Student. A Content Consistency Dashboard validates all simulation content. An External Grading Module allows instructors to grade student responses submitted via LMS platforms with AI-scored rubric feedback, bulk upload, grading history, and image attachment support (up to 5 per response); instructors can upload student-submitted Blackboard charts/visualizations alongside pasted essay text for GPT-4o vision grading, with bulk CSV grading supporting student-name-matched image uploads (e.g., `StudentName_1.png`). Attachment URLs are persisted in `grading_reports.attachmentUrls` (jsonb), displayed in results/history with paperclip indicators, and embedded in PDF exports. A Week Summary Dashboard provides visual distribution charts of class performance per week. A Student Feedback Survey collects per-week ratings and comments, stored in `survey_responses` table, with a Results Dashboard providing trend line charts, radar charts, and distribution charts. Server-rendered week pages include "Copy Answer Template" and "Download .txt" buttons alongside existing PDF download. A public-facing academic white paper, "Bridging the Relevance Gap: AI-Driven Experiential Learning for the Future of Work," synthesizes the platform's pedagogical foundations into 7 sections with academic citations. All PDF generators have been audited and fixed for proper text wrapping and page break logic. PDF rendering utilities are consolidated into shared modules: `client/src/lib/pdf-utils.ts` (TypeScript, for client-side PDF exporters) and `server/pdf-utils-inline.ts` (vanilla JS string via `getPdfUtilsScript()`, embedded in server-rendered `<script>` tags for weekly and grading pages). Shared functions include `addWrappedText`, `checkPageBreak`, `stripMarkdownForPdf`, `convertLatexToPlainText`, `convertMarkdownTables`, and `addWrappedMarkdown` (bold/header/formula-aware text renderer). The grading PDF strips markdown and LaTeX from AI-generated feedback fields before rendering. Both the White Paper and Institutional Proposal pages include comprehensive `@media print` CSS with A4 page sizing, reduced spacing, explicit card/table borders, font hierarchy (22pt/16pt/13pt headings), targeted `break-inside: avoid` on cards and reference entries, and print-only contact footers (hidden on screen via `display:none`, shown via `.print-only` class in print media) with Doug's contact info and confidentiality notices. Public-facing page CTAs are standardized, with educator-facing pages using "Request a Demo" and student-facing pages funneling toward sign-in/trial. SEO infrastructure includes GA4 SPA route tracking, Open Graph and Twitter Card meta tags, canonical links, a sitemap, `robots.txt`, JSON-LD structured data, and dynamic per-page document titles.

Security hardening includes Helmet.js middleware enforcing Content-Security-Policy (with CDN allowlists for jsPDF and KaTeX, and `script-src-attr 'unsafe-inline'` for server-rendered weekly page onclick handlers), Strict-Transport-Security (HSTS with preload), X-Frame-Options (DENY), X-Content-Type-Options, and Referrer-Policy headers. Tiered API rate limiting via express-rate-limit protects all endpoints (100 req/min global, 20/min auth, 30/min mutations). A cookie consent banner (Accept All / Essential Only) controls GA4 analytics loading, stored in localStorage. The privacy policy includes explicit GDPR and CCPA sections with lawful basis, data subject rights, international transfer disclosures, and a comprehensive data retention table. A Terms of Service page covers acceptable use, AI-generated content disclaimers, and Iowa governing law. Self-service data rights include JSON data export and account deletion requests from the user profile, backed by a `deletion_requests` table.
